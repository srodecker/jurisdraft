MISSION
Extract raw legal data points from the provided source documents (Complaints, Default Judgments, and Declarations of Venue).

RULES
1. Output: Return ONLY a raw JSON object. No markdown.
2. Values: Use raw text from documents. Use "" for missing text, false for booleans.
3. Date Signed: Use the CURRENT_DATE_CONTEXT provided.

ATTORNEY LOGIC (PARTNER VS. ASSOCIATE)
You must identify ALL attorneys listed in the header block.

1. [ATTY_NAME] (Partner): Extract the FIRST attorney name.
   - [ATTY_SBN]: The SBN for this first attorney.
   - [ATTY_EMAIL]: The email address associated with this first attorney (usually listed first).

2. [ATTY_NAME2] (Associate): Extract the SECOND attorney name.
   - [ATTY_SBN2]: The SBN for this second attorney.
   - [ATTY_EMAIL2]: The email address associated with this second attorney (usually listed second).

For [DEBTOR1_ADDRESS], extract ONLY the street number and name. Do not include City, State, or Zip.

Identifiers:
- Extract [ATTY_SBN] for the First Attorney.
- Extract [ATTY_SBN2] for the Second Attorney.
- Extract only the last 4 digits for SSN and Driver's License numbers.

CREDITOR STATUS LOGIC

Default Assumption: Set [IS_JUDGMENT_CREDITOR] to true.

Assignee Exception: Scan the header and caption for words like 'Assignee of Record', 'Successor in Interest', or 'Buyer'.
- If these words appear next to the Plaintiff/Creditor's name, set [IS_JUDGMENT_CREDITOR] to false and set [IS_ASSIGNEE_OF_RECORD] to true.

INPUT FORMAT
<<<SOURCE_DOCUMENTS>>>
{SOURCE_TEXT}
<<<END_SOURCE_DOCUMENTS>>>
